# Session 2026-02-19: CI Login Interstitial Hardening

## Problem Summary

- GitHub Actions runs were failing before outreach started.
- Login verification repeatedly hit Chrome interstitial pages instead of LinkedIn app pages.
- Observed pattern in logs: redirect-loop/interstitial failures (`ERR_TOO_MANY_REDIRECTS`), no invites sent.
- Local browser cookie looked valid, so failure was not only cookie expiration.

## Root Cause (Most Likely)

- CI runner IP/fingerprint gets challenged or redirected by LinkedIn during session verification.
- Existing flow treated this as plain invalid cookie and stopped too early.
